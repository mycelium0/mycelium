# Code of Conduct

## Our pledge

Mycelium is built by and for people who need dependable, private connectivity when networks are
unreliable. We pledge to make participation in this project a respectful, harassment-free experience
for everyone, regardless of background, identity, or experience level.

## Expected behaviour

- Be respectful, constructive, and patient; assume good faith.
- Focus on what is best for the project and its users.
- Give and accept feedback gracefully; disagree on technical merit, not on persons.
- Respect operator and user safety as the project's first requirement — never pressure anyone to
  weaken it for convenience.

## Unacceptable behaviour

- Harassment, insults, discriminatory or demeaning comments, or personal attacks.
- Publishing others' private information (an operator's identity, a node's address, a user's
  details) without explicit consent.
- Sustained disruption of discussion, or any conduct that would reasonably be considered
  inappropriate in a professional setting.

## Scope

This Code applies in all project spaces — issues, pull requests, discussions, and any space where
someone represents the project — and when an individual is officially representing the project in
public.

## Reporting & enforcement

Report unacceptable behaviour privately through a **[GitHub Security Advisory](https://github.com/mycelium0/mycelium/security/advisories/new)**
or by contacting the maintainers through GitHub. Reports are handled confidentially. Maintainers are
responsible for clarifying standards and may take any action they deem appropriate, from a warning to
a temporary or permanent ban from project spaces, and will respect the privacy and security of the
reporter.

This Code of Conduct is adapted in spirit from the [Contributor Covenant](https://www.contributor-covenant.org).
